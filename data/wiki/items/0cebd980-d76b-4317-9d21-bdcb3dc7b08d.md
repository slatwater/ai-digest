# dbskill：Claude Code 商业诊断技能包

## 概述

dbskill 是由 @dontbesilent 开发的 Claude Code 技能包，将作者 12307 条推文蒸馏为 4176 个原子知识单元，封装成 8 个可调用的诊断技能。覆盖商业模式验证、竞品分析、内容评估、小红书标题匹配等场景。

知识库完全开源，除了作为 Claude Code Skills 使用外，也可独立部署为系统提示词或用于 RAG 增强。

## 技能清单与用途

| 命令 | 功能 | 输入 | 输出 |
|---|---|---|---|
| `/dbs` | 智能路由入口 | 描述需求 | 自动分发到合适的子技能 |
| `/dbs-content` | 内容质量诊断 | 笔记/文案全文 | 五维质量诊断报告 |
| `/dbs-xhs-title` | 小红书标题生成 | 内容主题或笔记 | 从 75 个爆款公式中匹配推荐标题 |
| `/dbs-hook` | 开头钩子优化 | 开头文案 | 优化后的短视频/文章开头 |
| `/dbs-ai-check` | AI 痕迹检测 | 任意文本 | 22 条 AI 写作特征扫描结果 |

其余技能覆盖商业模式诊断、竞品分析等方向。

## 核心设计：知识蒸馏 + 公式匹配

dbskill 的方法论不是让 LLM 自由生成，而是**结构化知识驱动**：

- **知识蒸馏**：12307 条推文 → 4176 个原子知识单元，每个单元是一条可独立复用的经验规则
- **公式匹配**（以小红书标题为例）：内置 75 个爆款标题公式，输入内容后从公式库中匹配最适合的模板，而非纯生成。这种方式比自由发挥更稳定可控
- **技能间自动串联**：技能之间会主动推荐下一步，例如内容诊断发现「标题吸引力不足」时，自动建议调用 `/dbs-xhs-title`

## 小红书内容优化工作流

三步诊断链，形成闭环：

1. **`/dbs-content`**（内容诊断）→ 贴入笔记全文 → 输出哪里弱（如「开头缺钩子」「标题匹配度低」）
2. **`/dbs-xhs-title`**（标题匹配）→ 根据内容从 75 个公式中生成候选标题
3. **`/dbs-hook`**（开头优化）→ 优化视频/图文的开头钩子

也可直接从 `/dbs` 入口开始，描述需求后由系统引导走完整个链路。不需要用户做结构化处理，直接贴原始笔记即可。

**实战数据**：作者案例显示，使用该组合优化小红书内容后，3 小时内单篇突破 4 万阅读。

## 安装与部署

**Claude Code 内安装：**

```bash
npx skills add dontbesilent2025/dbskill
```

安装后在 Claude Code 对话中直接用斜杠命令调用。

**独立部署：** 知识原子文件可单独提取，作为系统提示词嵌入其他工具，或构建 RAG 知识库接入自有应用。
