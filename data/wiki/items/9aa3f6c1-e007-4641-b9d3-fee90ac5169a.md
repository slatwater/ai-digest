## 二、必须实现的数学工具函数

```glsl
// 1. 宏：smoothstep 别名
#define S(a, b, t) smoothstep(a, b, t)

// 2. 哈希：float -> vec3，用于给每个格子绑定随机参数
vec3 N13(float p) {
    vec3 p3 = fract(vec3(p) * vec3(.1031, .11369, .13787));
    p3 += dot(p3, p3.yzx + 19.19);
    return fract(vec3((p3.x+p3.y)*p3.z, (p3.x+p3.z)*p3.y, (p3.y+p3.z)*p3.x));
}

// 3. 哈希：float -> float
float N(float t) { return fract(sin(t*12345.564)*7658.76); }

// 4. 锯齿波：先平滑上升到b，再平滑下降到1
//    用途：驱动水滴的"积聚-滑落-复位"节奏
float Saw(float b, float t) {
    return S(0., b, t) * S(1., b, t);
}
```

**Saw 函数的行为说明**：`b=0.85` 时，`t` 从 0→0.85 输出值从 0 上升到峰值（水滴下滑），0.85→1.0 快速跌回 0（瞬间复位），模拟水滴积够重量才落下的物理节奏。

## 三、单层水滴核心函数 `DropLayer`

```glsl
// 返回值: vec2(折射偏移量, 该点是否在水滴/轨迹内)
vec2 DropLayer(vec2 uv, float t) {
    vec2 UV = uv;              // 保留原始uv用于wiggle计算

    // === Step 1: 格栅分区 ===
    // a.x=6 使格子横向密集（每行约12个滴），a.y=1 纵向稀疏
    uv.y += t * 0.75;          // 整体向上滚动（等效于滴向下）
    vec2 a    = vec2(6., 1.);
    vec2 grid = a * 2.;
    vec2 id   = floor(uv * grid);

    // === Step 2: 列间错位（消除人工整齐感）===
    float colShift = N(id.x);
    uv.y += colShift;
    id = floor(uv * grid);     // 错位后重算格子ID

    // === Step 3: 为此格子生成随机参数 ===
    vec3  n  = N13(id.x * 35.2 + id.y * 2376.1);
    // n.x → 水平初始偏移
    // n.y → 未使用（可扩展）
    // n.z → 时间相位 & 摆动方向

    vec2 st = fract(uv * grid) - vec2(.5, 0.); // 格内局部坐标，x居中

    // === Step 4: 计算水滴在格内的位置 p ===
    float x = n.x - .5;                        // x: [-0.5, 0.5] 随机初始位置

    // 蜿蜒路径：sin叠加产生S形摆动
    float y_world  = UV.y * 20.;
    float wiggle   = sin(y_world + sin(y_world));
    // 越靠近中心摆幅越大，随机决定左右偏向
    x += wiggle * (.5 - abs(x)) * (n.z - .5);
    x *= .7;                                    // 整体收窄，防止超出格边界

    // 锯齿波驱动y位置（0=顶部, 1=底部）
    float ti = fract(t + n.z);                  // 每滴相位不同
    float y  = (Saw(.85, ti) - .5) * .9 + .5;  // 映射到[0.05, 0.95]
    vec2 p   = vec2(x, y);                      // 水滴中心（格内坐标）

    // === Step 5: 椭圆SDF（竖向拉伸） ===
    // a.yx = (1, 6)：y方向×6，把圆变成竖向椭圆
    float d = length((st - p) * a.yx);

    // 主水滴轮廓（带柔边）
    float mainDrop = S(.4, .0, d);             // d<0.4开始有，d→0完全不透明

    // === Step 6: 拖尾轨迹 ===
    // 轨迹在水滴正上方，细而长
    float r    = sqrt(S(1., y, st.y) * S(.0, y, st.y)); // y范围限制
    float trailY = S(.5*r, r, abs(st.x - x));           // 水平贴近路径
    float trail  = trailY * S(-.02, .02, st.y - y);     // 只画水滴上方

    // === Step 7: 输出折射偏移 ===
    // 水滴/轨迹区域对背景UV产生扰动（产生折射感）
    vec2 offs = vec2(0.);
    offs += (mainDrop + trail * .3) * (p - st) * .3;

    return offs;
}
```

## 四、雾气层实现

```glsl
// 全屏雾气：用低频噪声对背景UV轻微模糊
// 实现思路：对uv加一层随机小偏移再采样背景
float StaticDrops(vec2 uv, float t) {
    uv *= 40.;
    vec2 id = floor(uv);
    vec3 n  = N13(id.x*107.45 + id.y*3543.1);
    // 只有部分格子"激活"，形成静止小水珠
    float fade = S(0., 1., fract(t + n.z) * 2.);  // 淡入淡出
    return S(.3, .0, length(fract(uv) - .5)) * fade * n.x;
}
```

## 五、多层叠加与主函数结构

```glsl
void mainImage(out vec4 fragColor, in vec2 fragCoord) {
    vec2 uv = (fragCoord - .5*iResolution.xy) / iResolution.y;
    float t  = iTime * .2;

    // --- 叠加3层不同速度/密度的雨滴 ---
    vec2 offs = vec2(0.);
    offs += DropLayer(uv * vec2(1., .5) + vec2(.1, t*.2), t);       // 层1：慢
    offs += DropLayer(uv * vec2(1., .5) + vec2(.2, t*.4), t*1.3);   // 层2：中
    offs += DropLayer(uv * vec2(1., .5) + vec2(.9, t*.3), t*.7);    // 层3：快

    // --- 雾气层 ---
    float fog  = StaticDrops(uv, t);
    offs      += vec2(fog) * .01;            // 雾对折射的微小贡献

    // --- 采样背景并应用折射偏移 ---
    vec2  refractedUV = uv + offs;
    vec4  bg = texture(iChannel0, refractedUV); // iChannel0: 背景图/视频

    // --- 后处理：暗角 + 色调 ---
    float vign = S(1.5, .5, length(uv));
    fragColor  = bg * vign;
}
```

## 六、关键参数速查表

| 参数 | 位置 | 含义 | 调节效果 |
|---|---|---|---|
| `a = vec2(6,1)` | DropLayer | 格栅密度 | x越大滴越密 |
| `0.75` | `uv.y += t*0.75` | 滚动速度 | 控制雨速 |
| `0.85` | `Saw(.85, ti)` | 滑落/复位比 | 越大下滑越慢 |
| `*.7` | `x *= .7` | 水平范围限制 | 防止滴出格边 |
| `20.` | `UV.y * 20.` | 蜿蜒频率 | 越大S形越密 |
| `.9+.5` | Saw 输出映射 | 滴的y范围 | 控制在[0.05,0.95] |
| `3层叠加` | mainImage | 层数 | 越多越自然越耗能 |

## 七、给实现 LLM 的注意事项

1. **没有粒子系统**——全部是逐像素 SDF 数学，不存在"创建水滴对象"的概念；
2. **折射而非透明**——水滴效果来自对背景 UV 的**偏移采样**，需要一张背景纹理 `iChannel0`；
3. **`Saw` 是核心节奏**——如果水滴运动不自然，首先检查 Saw 函数和 `ti` 相位的实现；
4. **ShaderToy 环境**：`iTime` = 时间，`iResolution` = 分辨率，`iChannel0` = 背景图；
5. **调试顺序**：先让一层 `DropLayer` 的 `d` 值可视化（输出 `vec4(d,d,d,1)`），确认椭圆 SDF 正确，再接折射和多层叠加。

## 非技术版：这个雨滴效果背后用了什么"底层技术"？

### 🖥️ 第一层：运行在显卡上的"像素程序"（Fragment Shader）

这个效果不是视频，也不是图片——它是一段**每秒运行数十亿次的小程序**，专门跑在你电脑的显卡（GPU）里。屏幕上每一个像素，它都单独计算一遍"这里该显示什么颜色"。这类程序叫 **GLSL Fragment Shader**，是实时图形渲染的底层语言。

### 🎲 第二层：用数学公式"伪造"随机（过程式噪声）

代码里的 `N13`、`N14`、`N` 函数，本质是用**数学公式模拟随机数**。真实世界里雨滴的位置、大小、速度都是随机的，但计算机不存真随机——作者用了一类叫**哈希函数**的数学技巧，输入一个位置坐标，就能输出一个"看起来随机"的值。这是程序化生成内容的核心底层思路。

### 📐 第三层：坐标变换（UV 空间操作）

屏幕被抽象成一张 0~1 的坐标网格（叫 **UV 空间**）。雨滴"向下滑动"的效果，实际上是不断**修改这个坐标系**，让纹理/颜色采样的位置随时间偏移，视觉上就像东西在动。

### 🌫️ 第四层：平滑插值（Smoothstep）

代码大量使用 `smoothstep`，这是一个**数学插值函数**，让颜色、边缘过渡得自然柔和——雨滴的边缘不会生硬，玻璃的雾气会渐变，都靠它实现。

---

**一句话总结**：这是一段跑在显卡上的数学程序，用坐标变换模拟运动、用哈希函数模拟随机、用插值函数模拟柔和——完全没有真实图片或物理模拟，**全是纯数学画出来的**。
