# Spark 2.0 — Web 端大规模 3D 高斯泼溅流式渲染引擎

## 一句话定义

Spark 2.0 是 World Labs 开源的 TypeScript 库，让浏览器能流畅渲染上亿级 3D 高斯泼溅（3DGS）场景。MIT 协议，基于 Three.js + WebGL2，Rust/WASM 混合实现。

## 解决什么问题

3D 高斯泼溅技术可以从多视角照片重建出逼真的 3D 场景，但数据量极大——一个高质量场景动辄几千万个 splat。消费级设备的浏览器通常只能处理 1-5M splat，超出就卡死或内存溢出。

Spark 2.0 的核心价值：**把「必须在高配电脑专业软件里看」变成「发个链接，任何人用浏览器流畅浏览」。**

## 核心技术架构

采用类似 Google 地图的「边看边加载」思路，由四个关键机制组成：

- **LoD Splat 树**：将原始 splat 逐层合并为粗到细的层级结构，渲染时用优先队列裁剪出当前视角最优的 50-250 万 splat 子集，帧率恒定
- **.RAD 格式**：自定义二进制格式，支持 HTTP Range 随机读取，首屏只需加载 64K splat 的粗略近似即可显示
- **GPU 虚拟内存池**：固定分配 16M splat 的显存池，用 LRU 页表在多文件间自动调度，避免显存溢出
- **两种 LoD 构建算法**：`tiny-lod` 在浏览器内秒级完成（适合快速预览）；`bhatt-lod` 用 Bhattacharyya 距离做离线高质量合并（适合生产环境）

## 使用门槛与成本

**完全免费开源（MIT 协议）**，可商用、可修改、无需注册。

技能要求：
- JavaScript / TypeScript — 必须
- Three.js 基础 — 需要了解
- Web 前端工程化（npm、打包工具）— 必须
- 3D 图形学 — 有帮助但非必须

本质上是 `npm install` 后几十行代码就能嵌入网页。关键前提是你得有 3DGS 数据（`.ply` 文件），可通过 Nerfstudio、gsplat 等开源工具从照片重建获得，再用 Spark 工具链转为 `.rad` 格式。

## 适用场景与目标用户

| 角色 | 场景 |
|------|------|
| Web 3D 开发者 | 在网站中嵌入高质量 3D 实景浏览（VR 看房、景点虚拟游览） |
| 3D 扫描从业者 | 将大规模扫描场景发布到网页，客户无需安装软件 |
| 游戏/元宇宙开发者 | Web 端基于真实场景的沉浸式体验 |
| 研究人员 | 在浏览器中展示和分享 3DGS 重建结果 |

**不适合**：没有 3DGS 数据需求的普通 Web 开发；AI Agent 无法直接调用（它是给人看的可视化工具，不提供语义查询接口）。

## 与 AI Agent 的关系

Spark 是纯前端渲染引擎，**Agent 本身用不了它**——没有后端 API，不提供场景语义查询。但 AI 编程助手（Claude、Cursor 等）可以辅助编写 Spark 集成代码、调试渲染问题、处理数据格式转换。

World Labs（李飞飞创办）的核心方向是空间智能（Spatial Intelligence），Spark 只是其开源的前端渲染组件，不代表 AI 能力本身。
