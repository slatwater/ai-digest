# 参考图采集 — 你的活儿（15 分钟）

**只放 3-5 张正面参考 + 1-2 张反面参考，不要更多。**

每张图旁边写一行注释（用 markdown 文件 `notes.md` 集中管理，或图片旁建同名 .txt）。

## 模板

```
references/
├── linear-sidebar.png
├── linear-sidebar.txt          ← 一行注释："喜欢这里的导航密度和键盘可达性"
├── stripe-docs-typography.png
├── stripe-docs-typography.txt  ← "喜欢这里的中英混排排印"
├── notion-database-card.png
├── notion-database-card.txt    ← "喜欢卡片堆叠的呼吸感"
└── bad-saas-template.png
    bad-saas-template.txt       ← "✗ 不想要：通用 SaaS 模板感、玻璃态滥用"
```

## 给 AIDigest 的备选方向（不一定全选，挑共鸣的）

如果暂时没头绪，可以从这些里挑灵感：

### 阅读型（适合 narrative 区域）
- **Stripe 文档**：中英混排、长内容可读性
- **Pitch.com 案例页**：编辑型杂志感
- **Linear changelog**：克制 + 留白

### 工具型（适合解析卡片、Wiki）
- **Linear Issue 列表**：信息密度 + 状态系统
- **Raycast**：键盘优先 + 元数据排版
- **Vercel Dashboard**：浅色 + 状态点 + 表格

### 研究型（适合实验、经验）
- **Are.na**：素材收集的卡片风格
- **Cosma / Logseq**：知识图谱的关系展示
- **Perplexity**：流式回答 + 来源标注

### 反面（明确不要）
- AI SaaS 通用首页（玻璃态卡片堆 + 渐变光晕）
- ChatGPT 早期界面（过于灰、没层次）
- Notion AI 的浮窗（颜色过亮、按钮拥挤）

## 抓图技巧
- 每张参考只截**你欣赏的那一块**，不要截整页
- 文件名带产品名 + 部位，例如 `linear-issue-list.png`
- 一张参考只对应**一个特征**，不要"我喜欢这张图所有东西"
