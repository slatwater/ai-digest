# constraints.md — 约束清单

## 响应式断点
**当前**：未做严格响应式（项目主要桌面端使用，开发期 1280-1440 优化）。
**重构请求**：
- **桌面优先**：1280px / 1440px / 1920px 都要好看
- **平板**：768-1024 可用即可（不强制完美）
- **移动**：暂不优化（用户场景不在手机上做研究）

## 暗色模式
**目前**：无暗色模式。
**重构方向**：可加，但**优先级低**——OKLCH 色板预留了暗色映射空间，但需要等浅色稿定下来再做。

## 国际化
- UI 全中文，不需要 i18n
- 但 mono 标注里大量英文技术名（`sonnet`、`opus`、`triage`、`Coze`）—— 中英文混排要好看，**字体回退栈很关键**

## 无障碍
- 键盘可达性：所有可点击元素都用 `<button>`，避免 div+onClick
- 焦点可见：`:focus-visible { outline: 2px solid var(--accent); outline-offset: 2px; }` 已设
- 颜色对比度：`text-primary` 在 `bg` 上 ≥ 18:1（OKLCH 12% on 100%）；`text-quaternary`（68%）只用于次要元数据

## 性能
- **不能引入大型组件库**（项目刻意保持零 UI 依赖）
- 动画用 CSS transition + transform，**不要**引入 framer-motion / react-spring
- 字体已用 next/font 本地化，**不要**换在线字体

## 内容长度边界
重设计时请确保以下极端情况不爆版：
- 解析卡片标题：最长可能 60 字（中英混合）
- 概念 chip：单个最长 30 字
- narrative：500-1500 字一段，含多个 [[]] 内嵌跳转
- 来源 URL：可能超过 100 字符，必须能截断
- 经验 markdown 内容：最长 5000+ 字，含表格 / 代码块 / 列表

## 流式状态（重要）
项目大量使用 SSE 流式输出。设计稿必须考虑：
- "正在生成"的视觉反馈（当前用三点脉冲 + 工具状态文字）
- 内容从 0 字逐渐增长到完整，**不能用骨架屏**（骨架屏假定知道最终高度）
- 工具调用提示（`正在搜索...` / `正在抓取...`）出现 1-2 秒后被替换为内容

## 颜色刚性约束
- `--accent`（青绿色）= "新知识/未读" 信号，重设计可调色相但**保留"新"=有色、"旧"=灰**的二元
- `--error`（红橙）= 错误，不要扩用到其他场景
- 不要引入第三种强调色（避免色彩噪音）
