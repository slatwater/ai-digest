# AIDigest — UI 重构 Brief

## 产品名
AIDigest

## 一句话定义
给 **AI 行业研究者 / 独立开发者 / 产品经理** 用的 **AI 前沿技术研究助手**，把每天散落的 AI 链接批量解析、深入研究、沉淀成可复用的知识库与实验经验。

## 核心用户
- **独立研究者 / 内容创作者**：每天处理 10–30 条 AI 推文/论文/GitHub 链接，需要快速辨真伪、识别新技术、积累私域知识
- **产品 / 技术决策者**：用它跟踪行业增量信号，识别可落地的技术

## Top 3 任务
1. 粘贴一批 AI 链接 → 等几分钟 → 拿回结构化解析卡片（含主角技术、一手溯源、增量价值判断）
2. 选感兴趣的卡片 → 深入提问对话 → 选区即存进 Wiki
3. answer 卡触发实验节点跑 Coze 验证 → 沉淀成「经验」（在 Wiki 内的经验 tab）

## 重构目标
**具体方向在 Claude Design 里现场对话。** 我会先看你出的初稿，再调方向 —— 不预设形容词，让 Claude Design 先放手。

## 不想要什么
- ❌ **无聊的、平庸的、传统 SaaS 风格** —— 玻璃态卡片堆 / 通用渐变光晕 / 千篇一律的「Get Started」按钮 / 圆角图标 + 截图拼贴的英雄区
- ❌ ChatGPT 早期那种过于灰白扁平、没有视觉锚点
- ❌ 暗色为主（项目偏阅读，浅色为基底）

## 技术栈
- **Framework**：Next.js 16（App Router）+ TypeScript
- **样式**：Tailwind CSS 4 + 自定义 OKLCH 色彩 token（详见 `docs/code-entry.md`）
- **组件库**：无（全部手写组件，没有 shadcn/MUI/Antd 包袱）
- **后端**：Claude Agent SDK + SSE 流式 + Python 抓取
- **Markdown 渲染**：react-markdown + remark-gfm

## 必须保留
- **OKLCH 色彩体系**：暖色调中性灰 + 墨绿强调（`accent: oklch(55% 0.15 192)`）—— 这是项目的视觉锚点
- **mono 字体作为元数据标注**：状态、token 计数、URL 域名都用 mono，不能换
- **ReactMarkdown 内容区域的可读性**：行高 1.7、prose 样式
- **3 个核心 view 的功能架构**：解析（统一画布）/ Wiki（含经验 tab）/ 运行原理（导航不能拆/合并）
- **SSE 流式反馈**：解析进度、深入提问、实验都是流式的，新设计要为「正在生成」状态留位置（不能纯静态稿）

## 可以推翻
- 卡片样式（解析卡片、Wiki 条目、经验条目）—— 全部可以重设计
- 留白节奏、卡片间距、字号层级
- 顶导样式（当前是文字 tab + 下划线）
- 空态插图 / 引导文案的位置
- 输入框、按钮的具体形态
- 折叠/展开的交互动画
- 状态标签的形态（当前用文字 + dot）
